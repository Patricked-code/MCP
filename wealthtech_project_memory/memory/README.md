# Mémoire projet WealthTech / MCP

Date de création : 2026-07-01
Dépôt : `Patricked-code/MCP`
Branche cible : `main`
Répertoire cible serveur demandé : `/root/wealthtech_project_memory/memory/`

Ce dossier contient la mémoire compilée de la conversation WealthTech / EWARI / KOREE / MCP / GitHub / Loop Engineering.

## Objectif

Centraliser dans le dépôt MCP les éléments de contexte nécessaires pour qu’un agent IA, Claude Code, ChatGPT, Codex ou un développeur humain puisse reprendre le projet sans repartir de zéro.

## Fichiers inclus

- `CONVERSATION_COMPILED_WEALTHTECH_EWARI_MCP.md` : compilation structurée de la conversation et des décisions.
- `MEMO_PROJECT_STABLECOIN_EWARI.md` : mémoire fonctionnelle, réglementaire et technique du projet EWARI / KOREE / XOF.
- `MEMO_MCP_GITHUB_SERVER_ACCESS.md` : état des accès MCP, GitHub, S1, S2, Claude et ChatGPT.
- `PROMPT_AUDIT_WEALTHTECH_MCP.md` : prompt d’audit non destructif à donner à Claude Code ou à un agent IA connecté au MCP.
- `INSTALLATION_MCP_MEMORY.md` : instructions pour installer/synchroniser cette mémoire sur le serveur dans `/root/wealthtech_project_memory/memory/`.
- `SUIVI_MEMORY.md` : point de reprise courant dédié à la mémoire projet.

## Règles de sécurité

- Ne jamais stocker de token complet dans ces fichiers.
- Ne jamais pousser `.env`, clés SSH privées, dumps SQL, archives privées, secrets GitHub ou secrets MCP.
- Toute action sur S1 ou S2 doit respecter la boucle Loop Engineering : lire la mémoire, inventorier, évaluer le risque, agir prudemment, tester, documenter.
